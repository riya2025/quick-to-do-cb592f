# Deploying Quick To-Do (Vercel + Render)

Generated by the **AI Foundry Deployment Agent**.

- **Frontend** -> **Vercel** (static Vite build, `vercel.json`)
- **Backend** -> **Render** Docker web service + managed **Postgres** (`render.yaml`)

## One-time setup

### Render (backend + database)
1. Push this repo to GitHub.
2. Render dashboard -> **New + -> Blueprint**, pick this repo. It reads `render.yaml`
   and creates the `quick-to-do-api` web service and `quick-to-do-db` Postgres instance.
3. `DATABASE_URL` is injected automatically from the database.
4. Copy the service's **Deploy Hook** URL into the GitHub secret `RENDER_DEPLOY_HOOK_URL`
   for push-to-deploy.

### Vercel (frontend)
1. `npm i -g vercel && vercel link` (or import the repo in the Vercel dashboard).
2. Add GitHub secrets: `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`.
3. Set the frontend's API base to the Render URL (e.g. `https://quick-to-do-api.onrender.com`).

## Push-to-deploy
`.github/workflows/deploy.yml` deploys the frontend to Vercel and triggers a Render
deploy on every push to `main`.
