# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

The app has a single Home screen where all task management happens.

**Home screen**

- **Top bar**: Displays the app name and a "Home" link to return to the main view.
- **Task input**: Type a task title into the "What needs to be done?" field and click **Add Task** (or press Enter) to create a new to-do.
- **Stats panel**: Shows counts for **Total**, **Active**, and **Done** tasks, updating automatically as you change your list.
- **Filter bar**: Three buttons — **All**, **Active**, and **Done** — each showing the count for that category. Click a button to show only matching tasks.
- **Task list**: Displays your tasks. Each task shows an auto-generated avatar, the task title, and action buttons:
  - **Done**: Marks an active task as completed (adds a "Completed" badge and grays out the title).
  - **Undo**: Marks a completed task as active again.
  - **Delete**: Removes the task from the list entirely.
- **Empty state**: When no tasks match the current filter, a message reads "No tasks here yet. Add one above!"

## Quick test

1. Open the app and confirm the Home screen loads with the task input field, empty stats (Total: 0, Active: 0, Done: 0), and the empty state message.
2. If the app is configured with a backend (`window.API_BASE`), open your browser's Network tab and confirm it successfully calls `/api/store/tasks` and receives a response.
3. Type a task title and click **Add Task**. Verify the task appears in the list, the Total and Active stats increment to 1, and the "All" filter pill shows 1.
4. Click **Done** on the new task. Confirm the title grays out, a "Completed" badge appears, the Active stat drops to 0, and the Done stat increments to 1.
5. Click the **Active** filter button and verify the completed task disappears; click **Done** and verify only the completed task shows; click **All** to see every task.
6. Click **Undo** to mark the task active again, then click **Delete**. Confirm the task is removed and all stats return to 0.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
