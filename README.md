# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

The app has a single screen with all controls visible at once:

- **Header**: Shows the app title ("Quick To-Do") and a badge displaying how many tasks are currently active.
- **New task input**: A text field labeled "What needs to be done?" where you type a task title.
- **Add button**: Click this (or press Enter) to add the typed task to your list.
- **Filter buttons (All / Active / Done)**: Three buttons that switch which tasks are visible. Each button shows a count of tasks in that category. "All" shows everything, "Active" shows incomplete tasks, and "Done" shows completed tasks.
- **Task checkbox**: Click the checkbox next to a task to toggle it between done (strikethrough text) and active (normal text).
- **Delete button (✕)**: Click the ✕ button on a task to remove it from the list permanently.
- **Completed summary**: When at least one task is marked done, a small line appears below the list noting how many tasks are completed.

## Quick test

1. Open the app and confirm the header displays "Quick To-Do" and the "0 active" badge.
2. Type a task title (e.g., "Buy groceries") into the input field and click **Add** (or press Enter); verify the task appears in the list and the active count updates.
3. Click the checkbox next to the new task; confirm the text gets a strikethrough, the active count decreases, and the "Done" filter count increases.
4. Click the **Done** filter button and verify only the completed task is visible; click **All** to see all tasks again.
5. Click the **✕** button on the task; confirm it disappears from the list and the counts update accordingly.
6. If a backend API is configured (`window.API_BASE`), open your browser's developer tools Network tab, add a new task, and confirm a successful POST request appears to `/api/store/tasks`.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
