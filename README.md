# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

The app has a single screen with all to-do features accessible at once.

**Header**
- Shows the app title "Quick To-Do" and a badge displaying the current number of active (incomplete) tasks.

**Add a task**
- **Text input**: Type the title of your new task here.
- **Add button**: Click this (or press Enter) to add the task to your list. The input clears automatically after adding.

**Filter tasks**
- **All / Active / Done buttons**: Click these to switch the view. "All" shows everything, "Active" shows incomplete tasks, and "Done" shows completed tasks. Each button displays a count of tasks in that category.

**Task list**
- **Circle/Checkmark button (○ / ✓)**: Click this on any task to toggle its status between active and done. Done tasks get a strikethrough and a green "Done" pill.
- **Delete button (✕)**: Click the red X to permanently remove a task from the list.

**Summary**
- Below the list, a status line shows how many tasks you have completed out of the total.

## Quick test

1. Open the app and confirm the header displays "Quick To-Do" and the "Loading tasks..." message disappears (if a backend is configured, confirm it responds and loads any existing tasks).
2. Type "Buy groceries" into the text input and click **Add** (or press Enter); confirm the task appears in the list and the input clears.
3. Click the circle button (○) next to "Buy groceries"; confirm it changes to a checkmark (✓), the text gets a strikethrough, and a green "Done" pill appears.
4. Click the **Active** filter button; confirm "Buy groceries" disappears from the list, then click **Done** and confirm it reappears. Click **All** to see all tasks again.
5. Click the red delete button (✕) next to "Buy groceries"; confirm the task is removed from the list entirely.
6. Refresh the page; confirm any remaining tasks persist (verifying that data is saved and reloaded correctly).

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
