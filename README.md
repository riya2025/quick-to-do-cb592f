# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

The app has a single main screen with three filtered views (All, Active, Done), accessible via the top navigation bar or the filter buttons within the card.

**Top Navigation Bar**
- **Quick To-Do (link):** Click the app title to return to the main "All" view.
- **All / Active / Done (links):** Click these to switch between viewing all tasks, only incomplete tasks, or only completed tasks.

**Task Input**
- **What needs to be done? (text field):** Type the title of your new task here.
- **Add (button):** Click this (or press Enter) to add the task to your list. The text field clears automatically after adding.

**Filter Bar**
- **All / Active / Done (buttons with pills):** These buttons also switch the task view. The numbered pill on each button shows the current count of tasks in that category.

**Task List**
Each task in the list displays a random image, its title, and action buttons:
- **Complete / Undo (button):** Click "Complete" to mark a task as done (it gets a "Done" badge and crossed-out title). Click "Undo" to mark a done task as active again.
- **Delete (button):** Click to permanently remove the task from the list.

If a filtered view has no tasks, an empty state message appears: *"No tasks here yet. Add one above!"*

## Quick test

1. Open the app and confirm the top navigation bar and the main card with the input field and filter bar load successfully.
2. Type a task name into the "What needs to be done?" field and click **Add**; verify the task appears in the list with a "Complete" button.
3. Click **Complete** on the new task; verify the title gets crossed out, a "Done" badge appears, and the button changes to "Undo".
4. Click the **Active** filter (in the top nav or filter bar); verify the completed task disappears from the list, then click **Done** to confirm it shows up there.
5. Click **Undo** on the task, then click **Delete**; verify the task is completely removed from the list.
6. If the app is configured to use a backend (`window.API_BASE` is set), open your browser's Network tab, add a new task, and confirm a `POST` request to `/api/store/tasks` returns a successful response.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
